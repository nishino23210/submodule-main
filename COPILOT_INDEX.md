# REFERENCE

このリポジトリは、メインリポジトリからサブモジュールを参照する構成を前提としています。`docs/` 配下の md ファイルは、サブモジュール側で管理される説明用ドキュメントであり、メインリポジトリからはその内容をまとめて参照する想定です。

## docs 配下の 3 種類の md ファイル

- `docs/program_base.md`: JBS 会社概要です。会社全体の考え方、事業イメージ、見方の整理をまとめています。
- `sub1/docs/program.md`: JBS の IR・株式メモです。公開情報ベースで、会社情報や投資家向けの見え方を短く整理しています。
- `sub2/docs/program.md`: 組織イメージと CAT 部の説明です。本部・部門の役割と、プログラム実装担当のイメージをまとめています。

## 使い分け

1. まず `docs/program_base.md` で会社全体の概要をつかみます。
2. 次に `sub1/docs/program.md` で IR・株式情報を確認します。
3. `sub2/docs/program.md` で組織の役割分担や実装担当の説明を確認します。
4. メインリポジトリでは、各サブモジュールを個別の更新単位として扱います。

## 注意点

- サブモジュール内の変更は、サブモジュール側でコミットと push を済ませたうえで、メインリポジトリで参照先を更新します。
- メインリポジトリだけを更新しても、サブモジュール内の実体ファイルは変更されません。
